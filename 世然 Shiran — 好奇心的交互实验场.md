# 世然 Shiran — 好奇心的交互实验场

## 项目定位

**世然**（Shiran）— 一个以 Explorable Explanations 为方法论前辈、面向中文用户的交互科普网站。通过 200 个探索单元，帮助用户理解世界的底层原理与经典思想实验。

> *世间万物皆有理，好奇一试便释然。*
>

### 本土化差异化（更适合中国用户）

- 选题本土化：从春运、外卖调度、短视频推荐、双十一博弈等场景切入底层机制。
- 叙事本土化：先讲现象再讲模型，术语同时给“白话解释 + 标准名词”。
- 体验本土化：移动端优先、竖屏友好、弱网可用，适配碎片化使用场景。
- 学习目标本土化：每个探索单元至少帮助用户带走一个可复述的原理。
- 分发本土化：与 B 站/微信生态联动，形成内容传播与回流闭环。

## 技术栈

| 层级 | 选型 | 理由 |
| --- | --- | --- |
| 框架 | **Astro 5 + SSG** | 内容优先 + Island Architecture，每个命题页零 JS 开销，模拟区按需水合 |
| 交互组件 | **Svelte 5** | 编译时框架，运行时 ~2KB（React ~40KB），与 Canvas 集成简洁 |
| 2D 渲染（默认） | **原生 Canvas 2D** | 无依赖，性能比 p5.js 高 3-5 倍，适配大部分画布型/揭秘型命题 |
| 2D 渲染（重型） | **PixiJS 8** | WebGL 加速，粒子系统/大规模 Agent 模拟场景使用 |
| 3D 渲染 | **Three.js** | 洛伦兹吸引子、三体等 ~15 个 3D 命题按需引入 |
| 数据可视化 | **D3.js** | 图表型命题标配 |
| 样式 | **Tailwind CSS 4** | 原子化 CSS，保持 200 页统一视觉 |
| 语言 | **TypeScript** | 类型安全，AI 辅助生成质量更高 |
| 内容管理 | **Astro Content Collections (MDX)** | 每个探索单元一个 `.mdx` 文件，叙事用 Markdown，交互用 Svelte 组件嵌入 |
| 部署 | **腾讯云（COS + CDN + API 容器服务）** | 统一国内可达性与后续扩展接口部署 |
| 包管理 | **pnpm** | 磁盘效率高，monorepo 友好 |

### 渲染层按模板分配

| 模板类型 | 渲染方案 | 典型命题 |
| --- | --- | --- |
| 🎨 画布型 | 原生 Canvas 2D / PixiJS（重型） | 康威生命游戏、Boids 鸟群、反应扩散 |
| ♟️ 对弈型 | Svelte 组件 + 少量 Canvas | 囚徒困境、最后通牒、纳什均衡 |
| 📊 图表型 | D3.js | 中心极限定理、Logistic 映射、分岔图 |
| 🔮 揭秘型 | Svelte 组件 + CSS 动画 | 蒙提霍尔、锚定效应、赌徒谬误 |
| 🌐 3D 场景 | Three.js | 洛伦兹吸引子、三体问题、Mandelbrot |

### 动态模型库原则（V1）

1. 不在主程序写死“200 条”。
2. 每个探索单元作为独立模块，最小单元为 `manifest.json`。
3. 主程序只读取 `model-index.v1.json`，索引由脚本从 `model_library/units/*/manifest.json` 自动构建。
4. 新增单元只需新增模块并重建索引，不需要改主程序常量。

### S/A/B 优先级规则（评分制）

| 维度 | 权重 | 评分说明（1-5） |
| --- | --- | --- |
| 顿悟强度 | 30% | 是否能稳定触发「原来如此」 |
| 交互可玩性 | 25% | 参数调整后反馈是否直观 |
| 复用潜力 | 20% | 引擎/组件是否可复用到其他命题 |
| 开发成本 | 15% | 成本越低分越高（低成本 = 5） |
| 传播潜力 | 10% | 是否具备分享和讨论价值 |

- `总分 >= 4.0` 且开发成本维度 `>= 3`：`S` 级（优先上线）
- `3.2 <= 总分 < 4.0`：`A` 级（次优先）
- `总分 < 3.2`：`B` 级（后续补充）
- 建议年度配比：`S=60`、`A=90`、`B=50`（合计 200）
